# api

在 src 文件夹下建一个 api 文件夹用于单独存放 api 的请求。然后分别导出各个请求，然后在业务代码里分别引入，这样的好处就是可以给每个请求命名一个语义化的名字。

先来看一下 api 的目录结构：

├── api                             // api 请求
│   ├── login.js                    // 登录请求
│   ├── userData                    // 用户数据模块请求

│   │   ├── type.js                 // 新增激活请求
│   │   └── statistics.js           // 付费统计请求

│   │   ├── aggregateStatistics.js  // 整体统计请求
│   │   ├── courseDetail.js         // 课程详情请求
│   │   ├── nps.js                  // NPS 评分请求
│   │   └── attendanceRate.js       // 到/完课率请求
│   ├── playback                    // 回放模块请求

api 的分类可以按照功能来分，也可以按照页面来分。因为本项目公用的 api 很少，基本是一个页面对应一个或两个 api 请求。所以这里是按照页面来分的。

我们来看几个示例

login.js

import request from '@request'   // 引入我们封装好的 request 方法

export function login(user_name, password) {
  return request({                                     // 返回一个 promise
    baseURL: process.env.VUE_APP_LOGIN_BASE_API,       // base_url
    url: '/auth/login',                                // 请求的地址
    method: 'post',                                    // 请求方式
    data: {                                            // post 请求的参数
      user_name,
      password,
      service_type: 'davinci',
    },
  })
}

api/premiere/nps.js

import request from '@request'

export function getNpsData(params) {
  return request({
    url: '/live/nps',
    method: 'get',
    params
  })
}

然后我们的请求统一用 store 进行管理，以首播 nps 接口为例。

src/store/modules/premiere.js

import { getNpsData } from '@api/premiere/nps'

const premiere = {
  ...
  actions: {
    ...,
    GetNpsData(context, params) {
      return getNpsData(params)
    },
    ...,
  },
}

export default premiere

在业务代码里这样使用：

src/views/premiere/nps/index.vue

...
<script>
import { Vue, Component } from 'vue-property-decorator'
import { namespace } from 'vuex-class'

const modulePremiere = namespace('premiere')   // 定义 premiere 模块

@Component
export default class NPS extends Vue {
  ...
  // 请求参数
  get params() {
    return {
      sku_mode: this.sku_mode,
      start_date: this.date_range.length && this.date_range[0],
      end_date: this.date_range.length && this.date_range[1],
      course_id: this.course_id,
    }
  }
  // 请求方法
  @modulePremiere.Action
  GetNpsData
  
  ...
  
  async _getNpsData() {
    this.loading = true
    try {
      const {
        data: { table_data },
      } = await this.GetNpsData(this.params)       // 发起请求
      this.tableData = table_data
    } catch (err) {
      console.log(err)
    }
    this.loading = false
  }
  ...
}
</script>

这里有个不成文的规定，为了方便管理，我在定义接口方法和模块的 Action 以及在业务组件里的请求方法它们的名字都是一样的，只不过形式不一样。在 api 里定义的方法用小驼峰命名（getNpsData）；在模块的 Action 里用大驼峰命名（GetNpsData）；在业务组件里的请求方法用下划线的形式命名（_getNpsData）。
