# 登录

登录的接口用 vuex 进行管理，对 token 和 user_id 进行了持久化。

整体的登录逻辑大概如下：

  1. 填写账号密码点击登录，进行表单验证，验证不成功给出提示；验证成功触发 Login action。
async handleLogin() {
  // 表单验证，validateInfo 为公用的验证方法
  const valid = await validateInfo.call(this, 'loginForm')
  try {
    if (valid) {                                            // 验证成功
      this.loading = true
      await this.$store.dispatch('Login', this.loginForm)   // 触发 Login
      this.loading = false
      this.$router.push('/')
    } else {                                                // 验证失败
      this.$message.warning('表单填写有误，请检查')
    }
  } catch (err) {
    console.log(err)
    this.loading = false
  }
}
  1. 请求接口，返回 token 和 user_id，分别存放在 cookie 和 store 里。
// src/store/modules/user.js
actions: {
  Login({ commit }, { username, password }) {
    username = username.trim()
    return new Promise((resolve, reject) => {
      ;(async () => {
        try {
          const {
            data: { token, user_id },
          } = await login(username, password)
          setToken(token)                                // 设置 token 的本地持久化
          setUserId(user_id)                             // 设置 user_id 的本地持久化
          setUsername(username)                          // 设置 username 的本地持久化
          commit(SET_TOKEN, token)
          commit(SET_USER_ID, user_id)
          commit(SET_USER_NAME, username)
          resolve()
        } catch (err) {
          reject(err)
        }
      })()
    })
  },
}
  1. 路由定位到 '/' this.$router.push('/')。
  2. 路由守卫导航开始进行判断，如果 cookie 里没有 token ，且不在白名单里就会返回到 login 页。
router.beforeEach((to, from, next) => {
  NProgress.start()
  if (getToken()) {
    ...
  } else {
    if (whiteList.includes(to.path)) {
      next()
    } else {
      next({ path: '/login' })
      NProgress.done()
    }
  }
})

  1. 如果 cookie 里有 token，就会携带 token 和 user_id 去请求 user_info 的接口去验证 token 的合法性。

  2. 如果 token 不合法，则请求不到数据，并给出错误提示 「验证失败，请重新登录」，并返回到 login 页。

if (getToken()) {
  if (to.path === '/login') {
    next({ path: '/' })
    NProgress.done()
  } else {
    if (!store.getters.name) {
      (async() => {
        try {
          await store.dispatch('GetInfo')
          next()
        } catch (err) {
          await store.dispatch('Logout')
          Message.error('验证失败，请重新登录')
          next({ path: '/' })
        }
      })()
    } else {
      next()
    }
  }
}
  1. 如果 token 合法，则请求到 user_info 数据，并触发 SET_USER_INFO 的 commit，将用户数据存放在 store 中。
// 获取用户信息
export function GetInfo({ commit }) {
  return new Promise((resolve, reject) => {
    (async() => {
      try {
        const { data: userInfo } = await getInfo()
        commit(types.SET_USER_INFO, userInfo)
        resolve()
      } catch (err) {
        reject(err)
      }
    })()
  })
}
